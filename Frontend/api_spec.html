API Spec CRS Server

/activate/0

This will tell the server that the channel 0 is now being listened on.

Responses:

Success

        {
            "success": "Channel was activated."
        }

Fail

        {
            "error": "Channel is already being listened on.",
            "error_id": 1
        }

/loose/0

This will tell the server that the channel 0 is no longer being listened on.

Responses:

Success

        {
            "success": "Channel was deactivated."
        }

Fail

        {
            "error": "Channel is not being listened on.",
            "error_id": 2
        }

/get/0

This will list all data the server has for the device listening channel 0.

Responses:

Success

        {
            "actions": [], // entries data in here is what was given via get args on /action/id
            "channel": 0
        }

Fail

        {
            "error": "Channel is not being listened on.",
            "error_id": 2
        }

/pending/0

This will list all pending actions the server has for the device listening channel 0.

Responses:

Success

        [] // entries data in here is what was given via get args on /action/id

Fail

        {
            "error": "Channel is not being listened on.",
            "error_id": 2
        }

/pop/0

This will give the client the oldest element in the action list and will remove it from the servers storage.

Responses:

Success

        {
            "next_action": {} // data in here is what was given via get args on /action/id
        }

No Actions Pending

        {
            "next_action": "idle"
        }

Fail

        {
            "error": "Channel is not being listened on.",
            "error_id": 2
        }

/action/0?cmd=send&text=hi

This will add an action to the queue for the device listening channel 0.

Responses:

Success

        {
            "action": {
                // data in here matches get args
                "cmd": "send",
                "text": "hi"
            },
            "success": "The action was added to the queue"
        }

Fail

        {
            "error": "Channel is not being listened on.",
            "error_id": 2
        }

/get

This will list all data the server has saved.

Responses:

Success

        [] // entries are the corresponding /get/id results.

/log/0?cmd=send&text=hi

This will add a log entry to the list of recoreded signals.

Responses:

Success

        {
            "action": {
                // data in here matches get args
                "cmd": "send",
                "text": "hi"
            },
            "success": "The action was added to the queue"
        }

Fail

        {
            "error": "Channel is not being listened on.",
            "error_id": 2
        }

/logs

This will show the list of recoreded signals.

Responses:

Success

        {
            "signals": [] // the format used with the get args
        }
